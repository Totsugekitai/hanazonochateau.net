### 自宅のルータを新調した

2025年まで自宅内のルータはYAMAHA RTX1200を使用していたが、いい加減古くなってきたし、メトリクスを取得するのにもSNMPとかいう面倒くさいものを使わなくてはいけなかったので、メンテする気力が失せてきた。
2026年になり、何かコンテンツを求めていた自分は、そういえば以前友人からプレゼントされていたTrend Micro CloudEdge SBがあったなと思い出した。

CloudEdge SBはUTMと呼ばれるセキュリティアプライアンス？みたいなやつらしいが、よくわからない。
ルータみたいな見た目をしている。
重要なのはこいつのスペックだ。

|    key    | value          |
|:---------:|:---------------|
|    CPU    | Celeron 4 core |
|  Memory   | 4GB            |
| Interface | RJ45(1GbE) x 5 |
|  Storage  | SATA SSD 32GB  |

けっこうリッチだ。
Debian入れて自前でルーティングさせるくらいはできそう。

さらにこいつ、ファンレスだ。
静かで自宅向きだ。

ということでDebian trixieをインストールして、 `netplan` でインターフェースを良い感じにして、 `/etc/nftables.conf` を書いて、 `dnsmasq` でDNSとDHCPをやらせてみた。
ネットワークのオタクではないので、IPv6はやっていない。

自分の使い方では今のところネットワークが詰まることはないし、けっこう満足している。
RTX1200は後輩に譲った。

### dotfilesを整理した

なんかコンテンツを求めていたので、dotfilesも整理した。
とりあえず以下を整理した。

- `.bashrc`
- `init.el`
- `.gitconfig`

`.bashrc` は主にプロンプトをいじった。

```
[hh:mm:ss]user@hostname:cwd(git:branch)
$
```

という形にしたかったので、 `PS1` をしばいた。
色々と落とし穴があったが忘れたのでAIとかに聞いてみてください。

`init.el` は一から作り直した。
`helm` とか使ってたけど、 `vertico` とか `marginalia` とか `corfu` とか、ナウいパッケージで書き直したので満足。

`.gitconfig` は今までdotfilesの管理下に置いていなかったが、置くようにした。

あとREADMEを置いて、管理にはGNU Stowを使うようにした。
もうシンボリックリンクを張るシェルスクリプトからはおさらばやね。
